## What was good
Lets say the vibe codign is when you do not write code 
Lets say tha AI Asisted coding is when you read (and understand!) all the code 

### I have really good understanding of what I want
Then using agent is basiacally no brainer. You are usually really quick to deremine if the result is correct or not. I would say that (depending on the amount of code written) the time saving here could be about 30%. It will not make you 10x engineer but still pretty good savings. The hard part here is the really good understanding. You do not get it with AI Agent. The struggle with the codebase and manually working on the problem is necessary (TODO - Articles) so find a time to get your reps in. But there are other type of work where this can happen.

### I want a simple tool and do not care about the code
Vibe coding FTW here. If it breaks it only affects me and simple scripts can save you so much time! Go for it
### I do not understand part of code 
And AI helps me to understand it. If it is some weird language construct during CR or some unknown language this is where AI shines.  
### I want to explore codebase
Again when I want to quickly find how something works the AI can do it more quickly than me using search (yes you can use grep sdkfj=śfojšč*** why not?). But I still try to verify the findings. 
### It made me better code reviewer 
Reading more code and the need to understand it made me better code reviewer. I tried to get better at this discipline for a longer period of time so this might be just coincidence. You really get you code reading reps in with AI and capturing the intent and looking for potential issues and things that smell is crucial skill.
### Checking my writing
Not writing, but checking my writing is really usefull. It is also great way to spot mistakes and blind spots in the writing. I usually use writing as tool for thinking (this article is no different) so pointing out gaps is really good. 
### Code review
For code review I think this could fall in really good and bad in the same time. It does not have any understanding of the intent so you usually get bunch of comments that does not really make any sense, but from time to time it saves your ass. So that is a good think. 

## Not so good

### I really want to understand something or I do not really know what I want 
Understanding of the code get hit. When you want to really understand something do not use AI to write your code struggle, be slow and just force yourself to wrestle with code. This have two benefits. First of all you will get better at coding and judging the code the machine gives you and also yhe amount of problems wehre you have really good understanding of what you want grows. If I do not know what I want I usually settle with something that looks good but I do not really understand. The speed benefit you still get is either eaten by the reading or (in the worst case) pushing something that does not work as should.
### Writing a DD 
Writing is tool for thinking. Forcing me to read something that AI wrote is rude. Do not do this. 
### Fancy setups 
I tried superhuman and few other fancy skills setup and was not impressed. Either I know what I want and simple planning mode is more than enough or you do not know what you want and you should discover it yourselve.:
